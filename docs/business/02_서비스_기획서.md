## 🚀 서비스 로드맵

### 1단계: 기본 서비스 (✅ 현재 완료)

```
사용자 플로우:
AI 원고 생성 → 사용자 확인 → 수동 다운로드 → 직접 포스팅
```

**제공 기능:**
- AI 원고 생성 (Gemini 2.5 Flash)
- 3회 재생성
- 메타데이터 개인화
- 원고 관리 (CRUD)
- TXT/PDF 다운로드

**사용자 인터페이스:**
- 대시보드 (사용량, 공지사항)
- 원고 생성 폼
- 원고 목록 (필터링, 검색)
- 프로필 관리
- 구독 관리

### 2단계: 완전체 서비스 (🎯 2~3개월 후)

```
사용자 플로우:
AI 원고 생성 → 사용자 승인 → 워드프레스 자동 발행 → 검색 최적화 모니터링
```

**추가 기능:**
- 워드프레스 자동 발행
- 시간차 예약 포스팅
- SEO 자동 최적화
- AEO 대응 (AI 검색 최적화)
- GEO 대응 (퍼플렉시티/ChatGPT 최적화)
- 성과 분석 대시보드

**사용자 선택권:**
- 자동 모드: AI가 모든 것 처리
- 수동 모드: 직접 편집 가능

## 🔧 기술 구성

### 핵심 기술 스택

```
Frontend: React + Material-UI
Backend: Firebase Functions (Node.js)
Database: Firestore
Storage: Firebase Storage
Auth: Firebase Auth (네이버 로그인)
Payment: 네이버페이
AI: Google Gemini 2.5 Flash
```

### AI 원고 생성 로직

```javascript
최종 원고 = Gemini_API (
  기본 페르소나
  + 정치 가이드라인
  + 사용자 프로필
  + 사용자 입력값
  + 실시간 뉴스 요약
)
```

### 메타데이터 개인화 시스템

**저장 데이터:**
```javascript
{
  userId: "user123",
  preferences: {
    selectedStyles: ["친근함", "논리적"],
    favoriteTopics: ["교육", "교통"],
    recentSelections: [...] // 최근 선택한 원고 패턴
  },
  district: "서울 강남구",
  position: "구의원"
}
```

**학습 방식:**
- 사용자가 선택한 원고 패턴 추적
- 재생성 시 이전 선호도 반영
- 3~5회 사용 후부터 개인화 효과 발생

## 👥 사용자 경험 설계

### 온보딩 프로세스

```
1. 네이버 로그인
   ↓
2. 당원 인증 (당원증 OCR)
   ↓
3. 프로필 입력 (지역구, 직책, 정치 성향)
   ↓
4. 요금제 선택 (스탠다드 플랜)
   ↓
5. 결제 (네이버페이)
   ↓
6. 서비스 시작
```

### 원고 생성 플로우

```
1. 대시보드에서 "원고 생성" 클릭
2. 입력 폼 작성 (주제, 키워드, 톤앤매너, 지시사항)
3. "생성하기" 버튼 클릭
4. 로딩 (10~20초)
5. 원고 초안 1편 제시
6. 불만족 시 재생성 (최대 3회, 이전 초안 보관)
7. 최종 확정
8. 다운로드 (TXT/PDF)
```

### 대시보드 구성

**주요 지표:**
- 이번 달 사용: X / 90회
- 남은 횟수: X회
- 다음 갱신일: YYYY-MM-01

**빠른 액션:**
- 원고 생성하기
- 최근 원고 보기
- 키워드 분석 (2단계)

**공지사항:**
- 서비스 업데이트
- 정치 관련 알림
- 선거법 주의사항
